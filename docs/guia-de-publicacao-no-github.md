# Guia rápido de publicação no GitHub

## Estrutura recomendada

- manter o `README.md` na raiz sempre atualizado;
- usar títulos padronizados nas pastas e arquivos;
- versionar mudanças pedagógicas com mensagens de commit claras;
- registrar revisões importantes antes de apagar conteúdo antigo.

## Sugestão de rotina da equipe

### Antes da semana começar
- revisar o plano semanal;
- ajustar lista e cartilha;
- subir os arquivos revisados;
- registrar observações no commit.

### Depois da aula
- anotar o que funcionou e o que precisa melhorar;
- marcar dificuldades recorrentes da turma;
- atualizar o plano da semana seguinte, se necessário.

## Padrão de mensagem de commit

- `docs: atualiza readme do eixo de matemática`
- `matematica: ajusta semana 07 com reforço em sistemas lineares`
- `avaliacao: adiciona mini simulado 02`
- `cartilha: revisa cartilha 04 de geometria plana`
